Test 1 — Android 12: Audio plays after plan generation (pre-unlock fix)

Simulates the full runPipeline() path: two real network awaits (/scan-and-plan, /tts) that expire the user-gesture token on Android 12, followed by a play() call on a pre-unlocked Audio element (the fix).

Before fix: new Audio(blob) created after 4 async hops → NotAllowedError, audio unavailable.
After fix: new Audio() created + play-paused synchronously in the click handler → token captured → real audio plays when blob arrives.

Test 2 — Android 16: cmd listener recovers after recognition cool-down

Calls startCmdListener() while simulating recognition.start() throwing InvalidStateError (Android Chrome recognition service still cooling down).

Before fix: catch(_){} silently consumed the error; ttsListening stayed true with recognition dead → all future voice commands ignored.
After fix: catch rolls ttsListening back to false and schedules a rearm → listener self-heals.
