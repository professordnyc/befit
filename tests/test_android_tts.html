🔬 Befit - Android TTS Regression Tests

Run in Chrome on the target Android device. Each test isolates one bug. A timestamped event log appears at the bottom.

Test 1 — Android 12: play() rejection must not be silent

Simulates ttsPlay() being called after four chained awaits (matching the real /scan-and-plan + /tts fetch chain). On Android 12 Chrome the user-gesture token expires before .play();
before fix: NotAllowedError silently discarded, UI freezes.
after fix: await ttsAudio.play() routes the rejection to the catch block → “⚠️ Audio unavailable” shown (or audio plays if policy permits).

Step: Tap Run Test 1 — that is the only action needed.

Test 2 — Android 16: resume after pause must use .src

Loads audio, pauses it, then exercises the patched resume guard.
before fix: guard checked .currentSrc, which is "" on Android Chrome → fell through to a redundant /tts re-fetch (audio never resumed).
after fix: guard checks .src, which is always set at new Audio(url) construction → correctly resumes the paused element.

Steps: 1 Load & Play  →  2 Pause  →  3 Resume.
